Ты — опытный Python/Django разработчик и наставник. Сгенерируй полный проект Django (уровень junior, но “как в проде”) под названием `skill_blog`.
Цель: я хочу изучить, как ты реализуешь Django-проект, поэтому код должен быть читабельным и снабжён ОБЪЯСНЯЮЩИМИ комментариями на русском (не формальными, а реально полезными).

ОБЯЗАТЕЛЬНО:
1) Пиши комментарии на русском в ключевых местах: модели, формы/валидация, permissions, views, urls, шаблоны, настройки. Комментарии должны объяснять “почему так”, а не “что это”.
2) Дай пошаговые команды для запуска проекта локально (venv, pip install, migrate, createsuperuser, runserver).
3) Используй Django 5.x. База: SQLite (по умолчанию).
4) Всё должно быть в одном репозитории. Итог — рабочий проект без заглушек.

ФУНКЦИОНАЛ ПРОЕКТА (Уровень 1):
A) Кастомный пользователь:
- Создай кастомную модель пользователя на базе AbstractUser (приложение `accounts`).
- Добавь профиль пользователя: avatar (ImageField, можно хранить локально), bio (TextField), created_at.
- Реализуй регистрацию, логин, логаут.
- Страницы: /register, /login, /logout, /profile/<username>.

B) Блог:
- Приложение `blog`.
- Модели:
  - Post: title, slug, content, author(FK->User), created_at, updated_at, is_published(Boolean), category(FK)
  - Category: name, slug
  - Comment: post(FK), author(FK), text, created_at, is_approved(Boolean)
- Требования:
  - Список постов: показывать только опубликованные (is_published=True). Пагинация 10/стр.
  - Детальная страница поста по slug.
  - Создание/редактирование/удаление поста:
    - Создавать может любой залогиненный.
    - Редактировать/удалять может только автор (строгая проверка).
  - Комментарии на странице поста:
    - Залогиненный может оставить комментарий.
    - Комментарий попадает на модерацию (is_approved=False).
    - На странице поста показывать только одобренные комментарии.
  - Поиск: простая строка поиска по title и content (GET параметр q) на странице списка постов.
  - Категории: страница категории /category/<slug> со списком опубликованных постов данной категории.

C) Админка:
- Настрой admin.py для Post/Category/Comment/Profile:
  - list_display, search_fields, list_filter
  - Автоматическое заполнение slug из title/name (prepopulated_fields)
  - Inline для комментариев на странице поста в админке
  - Удобная модерация комментариев (bulk action “approve comments”)

D) UI (минимально приятный):
- Используй Django templates + базовый layout (base.html).
- Навигация: Главная, Категории, Создать пост (если залогинен), Профиль, Войти/Выйти.
- Простая верстка (без сложных CSS, можно минимальный стиль в static).

E) Безопасность и качество:
- Используй CSRF токены в формах.
- Используй LoginRequiredMixin/permission checks.
- В views избегай N+1 запросов: используй select_related/prefetch_related там, где нужно.
- Обработай 404/403 нормально (шаблоны ошибок можно простые).

СТРУКТУРА:
- Django project: skill_blog/
- apps: accounts, blog
- templates разложить по папкам
- static/ и media/ настроить корректно
- requirements.txt
- README.md с описанием и командами запуска

2) В конце дай чеклист для ручной проверки: регистрация, создание поста, модерация комментариев, права доступа, пагинация, поиск.
